## üèóÔ∏è 1. Core Architecture (The Skeleton)

The application is divided into a robust logic core and a generic presentation layer.

### üß† 1.1 `src/core/` (Logistics & Intelligence)
Contains the engineering rules and communication protocols. **Zero UI components allowed here.**
*   **`bridge/`**: Handles the communication membrane (`CoreBridge.js`) and contract discovery (`ContractSensing.js`).
*   **`state/`**: Centralized reactive state management (Zustand). Stores contracts, nodes, and connections.
*   **`integrity/`**: Validation and sanitization rules (`Sanitizer.js`, `ContractIntegrity.js`).
*   **`diagnostics/`**: Real-time telemetry parsing (`DiagnosticParser.js`).
*   **`projection/`**: High-performance rendering motor (`ProjectionKernel.js`). **Interface for the Presentation Layer.**

### üé≠ 1.2 `src/presentation/` (Generic Rendering)
The physical manifestation of data. It projects shapes based on instructions from the `core`.
*   **`MethodInvoker.jsx`**: Generic executor for any backend capability.
*   **`SystemRegistry.jsx`**: Visual directory of project artifacts.
*   **`SchemaResolver.js`**: The Translation Kernel that bridges Roles to Widgets.
