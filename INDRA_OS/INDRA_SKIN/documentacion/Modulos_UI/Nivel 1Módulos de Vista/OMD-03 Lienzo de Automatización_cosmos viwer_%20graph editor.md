# Blueprint Maestro OMD-03: Lienzo de Automatización (Axiomatic Fractal Flow)

## 1. Identificación y Alcance (ID & Context)
- **ID Técnico:** `view_fractal_orchestrator`
- **Nombre Funcional:** Lienzo de Automatización (The Continuum).
- **Naturaleza:** Escenario Principal de Nivel 1 (Macro-Cosmos).
- **Axioma de Diseño:** "El orden no se crea, se descubre. El flujo no se arrastra, se habita."

## 2. Anatomía del Continuum (Universal Layout)
El OMD-03 abandona la rejilla estática por un **Autolayout Axiomático Fractal**. La organización espacial es un **resultado** de la lógica de datos, no una decisión del usuario.

### A. El Sistema de Columnas Infinitas (Linear-Fractal Flow)
- **Crecimiento Orgánico:** Los nodos se organizan en "Columnas de Resonancia" que fluyen de izquierda a derecha.
    - **Siniestra (Seeds):** Orígenes de datos (Vaults, Triggers).
    - **Medium (Transforms):** El "Cuerpo" de la lógica (Math, AI, Filters).
    - **Diestra (Fruits):** La manifestación final (Slots, Exports, Dispatchers).
- **Fractalidad:** Cada columna puede subdividirse en sub-flujos si la complejidad de la tarea lo requiere, creando un "árbol de decisión" visualmente coherente.

### B. El Mecanismo de Atención (Focus & Depth)
- **Escalabilidad de Profundidad:** El sistema opera bajo un modelo de "Lentes de Realidad".
    - **Macro-View:** Vista general del grafo donde los nodos son abstractos y simplificados.
    - **Meso-View (Focus Mode):** Al interactuar con un nodo (Doble Click), el sistema aplica el motor de la esfera (Blur + Depth Layer). El nodo se expande y revela su terminal o editor interno.
    - **Micro-View (Sub-Atom):** Dentro de un "Nodo Compuesto" (ej: Slot), se despliegan herramientas específicas que antes estaban ocultas.

## 3. Definición Funcional (Lógica Celular)

### A. Manifestación Predictiva (Axiomatic Sieve)
- El catálogo de componentes (OMD-08) es dinámico. Si el Cosmos está vacío, solo se permiten "Semillas". 
- A medida que el flujo crece, el sistema desbloquea "Arquetipos Concomitantes" basados en los puertos disponibles.

### B. El Hilo de Ariadna (Cables Inteligentes)
- Los cables no son solo líneas; son **vasos sanguíneos**. 
- **Tensión Sináptica:** La distancia entre nodos se ajusta automáticamente para minimizar el "ruido visual" de los cables.
- **Validación Biológica:** Los puertos solo aceptan conexiones que mantengan la integridad del tipo de dato.

## 4. El Nodo Compuesto (Andamiaje de Propiedades)
Los nodos no son ventanas aisladas, son estaciones de un mismo proceso.
- **Herramientas de Diseño:** No existen como nodos independientes en el canvas. Son sub-módulos que "viven" dentro de un nodo de alta jerarquía (como el Slot Node).
- **Herencia Automática:** Si un nodo hereda la propiedad "VISUAL", desbloquea automáticamente sub-herramientas de color, timeline y máscara.

## 6. Auditoría de Ergonomía Cognitiva
- **Eliminación del 'Loco dragging'**: El usuario ya no organiza nodos; el usuario conecta intenciones. El sistema garantiza la limpieza visual.
- **Revelación Progresiva**: Evitamos el parálisis por análisis ocultando herramientas complejas hasta que el nodo tiene el arquetipo necesario para usarlas.
