# Andamiaje Canónico: El Continuum de Manifestación (ISK v9.0)

## 1. El Axioma de la Profundidad
> *"La complejidad no se muestra, se habita. La interfaz no se diseña, se revela."*

El sistema opera bajo un andamiaje jerárquico-fractal donde cada nivel de "Foco" desbloquea un nuevo set de leyes y capacidades.

---

## 2. Niveles de Manifestación

### L1: MACRO-COSMOS (El Orquestador)
- **Rol:** Visión sistémica lineal-fractal.
- **Dharma:** Conexión de intenciones.
- **Herramientas:** Nodos abstractos, cables, auto-layout columnar.
- **Limitación:** Las propiedades detalladas están ocultas ("folded").

### L2: MESO-FOCUS (El Artefacto Activo)
- **Rol:** Intervención específica. Se activa por Doble Click.
- **Dharma:** Configuración y lógica de negocio.
- **Expresión:** El nodo se expande, el entorno se desenfoca (Layer Blur). Aparecen terminales, tablas de datos o editores simples.

### L3: MICRO-SUBATOM (El Nodo Compuesto)
- **Rol:** Creación de realidad específica (ej: Slot Editor, 3D Scene).
- **Dharma:** Diseño HiFi, manipulación de baja latencia.
- **Expresión:** Desbloqueo de sub-paneles y arquetipos especializados.

---

## 3. Lógica Celular de Arquetipos (Herencia Adjetiva)

El sistema de descubrimiento sigue una ley de **Herencia Prohibida**: Un sub-arquetipo no puede manifestarse si su arquetipo "Padre" no está presente o no tiene alimentación de datos.

### Caso de Estudio: Adapter "3D Animation" (Puntos de Fricción)
1.  **Carga del Contrato:** El Core persiste el JSON del Adapter con arquetipos base: `[VISUAL, TIMELINE, SPATIAL_3D]`.
2.  **Manifestación en Front:**
    - El `ComponentProjector` detecta **VISUAL** -> Inyecta puerto de Color, Materiales y Máscaras.
    - Detecta **TIMELINE** -> Inyecta puerto de Keyframes y Curvas de Animación.
3.  **Fricción Lógica (Seguridad Semántica):**
    - Si el nodo pierde la conexión a una **Semilla Visual**, los sub-nodos de "Color Editor" o "RGB Curves" se vuelven **Gris (Inactivos)**. La UI no desaparece (para evitar saltos), pero la lógica "muere" visualmente.

### Diagrama de Flujo de Herencia:
`TRUNCAMIENTO -> SEMILLA -> ADAPTER [PADRE] -> [SUB-HERRAMIENTAS]`
- Si `SEMILLA == NULL` -> `ADAPTER == OFFLINE` -> `SUB-HERRAMIENTAS == HIDDEN/VOID`.

---

## 4. Auditoría de Fricción y Fallos Extremos

**Punto de Fricción A: La Explosión de Propiedades.**
Si un Adapter como Blender trae 50 sub-arquetipos, la UI podría colapsar.
- **Solución:** Agrupación por **Intención**. Solo se revelan los sub-arquetipos que tienen un puerto de entrada activo o que el usuario "llama" en el modo foco.

**Punto de Fricción B: La Bimodalidad de Datos.**
¿Qué pasa si la lógica pide un render 3D (HiFi) pero el sistema está en modo LoFi?
- **Solución:** El andamiaje permite el **Proxy Visual**. El front visualiza una representación simplificada (bounding boxes) mientras el motor HiFi procesa la realidad en segundo plano.

---

## 5. Conclusión Axiomática
El andamiaje es agnóstico al contenido. No importa si es un editor de facturas o un motor 3D; si el artefacto declara su identidad mediante arquetipos, el sistema proyecta la interfaz necesaria de manera bimodal y fractal.
