# WORKSPACE_MANIFEST: EL MODELO TRÍPTICO ÁUREO (v2.0)

## 1. COMPOSICIÓN GEOMÉTRICA (Φ)
El viewport se rige por la Proporción Áurea para optimizar la carga cognitiva:
*   **Columna I (Potencia)**: `28%` del eje X. Buffer de reserva de materia prima.
*   **Columna II (Agencia)**: `44%` del eje X central. Núcleo de atención y operación.
*   **Columna III (Manifestación)**: `28%` del eje X. Buffer de salida y resultados.

## 2. AXIOMAS DE FRACTALIZACIÓN
Queda prohibido el uso de una barra de herramientas global para la creación de contenido. La invocación es local y específica:
*   **Gatillo de Siembra (Col I)**: Botón `SEED_POTENCY`. Invoca exclusivamente flujos de creación de `DATA_SCHEMA`.
*   **Gatillo de Ignición (Col II)**: Botón `IGNITE_AGENCY`. Invoca flujos de `BRIDGE` y `WORKFLOW`.

## 3. PROTOCOLO DE AGENCIA (DIFERENCIACIÓN DE ACCIÓN)
El `AgencyChassis` debe proveer dos vías de interacción deterministas:
1.  **Gatillo PULSE (Micro-Acción)**: Ejecución atómica en segundo plano. No cambia el viewport. Indica progreso via Step-Track visual.
2.  **Gatillo FOCUS (Macro-Acción)**: Navegación profunda a Nivel 3 (Macro Engine).

## 4. ESTÉTICA HUD Y RUIDO COGNITIVO
*   **Contadores de Densidad**: Obligatorios en el encabezado de cada columna (ej. `[ 12 ]`).
*   **Highlight de 3 Vías**: 
    *   `selective`: Resalto cian para el foco directo.
    *   `relational`: Resalto violeta para conexiones semánticas.
    *   `dimmed`: Atenuación pasiva para el resto del ecosistema.
*   **Scroll Silent**: Uso mandatorio de la clase `.no-scrollbar` para mantener la pureza del tríptico.

## 5. RESTRICCIONES TÉCNICAS
*   **Variables de Espacio:** Solo usar `var(--indra-ui-margin)` y la escala `var(--space-X)`.
*   **Variables de Color:** Prohibido el uso de valores `px` o `color` fuera de `design_tokens.css`.
*   **Gatillos**: Toda acción destructiva DEBE usar componentes con protección `hold`.
